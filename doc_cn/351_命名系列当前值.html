# 命名系列当前值

命名系列功能允许你为文档命名定义前缀。例如，如果销售订单的前缀为"SO"，则系列将生成为 SO-00001、SO-00002... 依此类推。点击此处了解如何在 ERPNext 中自定义交易/主数据的编号系列。

1. 设置当前值

命名系列功能还提供了一个工具，你可以为特定前缀设置当前值。如果你最近开始使用 ERPNext，并且在旧系统中有历史交易记录，而你想让编号系列从旧系统的结束位置继续，通常需要用到这个功能。让我们通过一个场景来更好地了解这个功能。

例如，你在旧系统中创建了 322 个销售订单，其中 SO00322 是最高销售订单 ID。在 ERPNext 中，你需要第一个销售订单在保存时自动获取 #323。为此，你需要按以下步骤为 SO 系列设置当前值。

进入命名系列工具

设置 > 系统 > 命名系列

更新系列部分

!图片

选择前缀

根据我们的场景，销售订单的前缀为"SO"。

!图片

当前值

如果你当前账户中已创建了 12 个销售订单，那么更新的当前值将为 12。你可以将当前值编辑为 322，然后点击"更新系列编号"。

!图片

通过此设置，新销售订单的编号将从 #323 开始。

2. 系列编号错误

如果在保存交易时收到重复名称错误，例如在保存 Item Price 时收到以下错误：

Duplicate name Item Price RFD/00016

此错误消息表明，当你保存 Item Price 时，系统尝试为该 Item Price 记录分配"RFD/00016"。但系统发现该 ID 的 Item Price 已存在于你的系统中。

此错误可能是因为 Item Price 的系列/前缀当前值被扰乱，与实际当前值不同步。Item Price 的实际当前值可能是 20（或任何大于 16 的数字），但有人将此系列的当前值设置为了 15。

要确认特定系列的实际当前值，你应该检查相关文档（此处为 Item Price）的报表，并查找具有最高值的 Item Price ID。

假设我们发现 Item Price 的实际当前值为 22，那么你可以进入命名系列，将 Item Price 前缀/系列的当前值设置为 22，然后更新系列编号。

这些说明适用于 ERPNext 中所有用户可以自定义系列及其当前值的文档。

让我们再考虑另一个场景来更好地了解。在将文档分配给另一个用户时，错误消息显示：

Duplicate name ToDo TDI00014286

这表明 ToDo（TDI）的系列/前缀当前值已被扰乱。你应该按照以下步骤更正 TDI 前缀的当前值。

- 在 ToDo 报表中检查最高的 ToDo ID 值。 - 设置 >> 设置 >> 命名系列 - 检查更新系列的 B 部分 - 为 ToDo 选择前缀"TDI" - 确保 ToDo 的最高编号作为当前值在命名系列中更新。如果没有，请更正当前值，然后点击"更新系列编号"。
