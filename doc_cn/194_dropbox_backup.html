设置 Dropbox 备份

我们始终建议客户在 ERPNext 中维护其数据的备份。数据库备份以 SQL 文件的形式下载。如果需要，这个备份 SQL 文件也可以恢复到另一个 ERPNext 账户。

你可以将 ERPNext 账户的数据库备份自动下载到你的 Dropbox 账户。

注意：如果你是 Frappe Cloud 用户，现场和异地备份会自动为你创建：<https://frappecloud.com/docs/sites/backups>

要设置 Dropbox 备份，请进入 > Home > Integrations > Dropbox Settings

步骤 1：登录 Dropbox 开发者区域

<https://www.dropbox.com/developers/apps>

步骤 2：创建新的 Dropbox 应用

步骤 3：填写新应用的详细信息

步骤 4：插入你的自定义域名 Redirect URI

https://{yourwebsite.com}/api/method/frappe.integrations.doctype.dropbox_settings.dropbox_settings.dropbox_auth_finish

步骤 5：在新窗口中，打开你的 ERPNext 安装中的 Dropbox Settings 页面

步骤 6：设置备份频率和电子邮件

设置将站点备份下载到 Dropbox 账户的频率。

步骤 7：从你的 Dropbox 应用窗口输入密钥

从你的 Dropbox 应用页面，将应用密钥和（未隐藏的）应用密钥输入到 ERPNext Dropbox 设置页面。

或者，你可以将其手动输入到 sites/{sitename}/site_config.json 中，如下所示：

       {
	 "db_name": "demo",
	 "db_password": "DZ1Idd55xJ9qvkHvUH",
	 "dropbox_access_key": "ACCESSKEY",
	 "dropbox_secret_key": "SECRECTKEY"
}

步骤 8：继续之前点击保存

步骤 9：保存后，点击"Allow Dropbox Access"

Dropbox 登录页面将在新标签页中打开。这可能需要你允许弹出窗口访问你的 ERPNext 账户。

步骤 11：允许 Dropbox 访问

成功登录后，你会看到如下确认消息。点击"Allow"以允许你的 ERPNext 账户访问你的 Dropbox 账户。

步骤 12：确认备份工作正常

从 ERPNext Dropbox 页面，点击Take Backup Now，然后转到你的 Dropbox 文件视图。你应该在 Dropbox 中看到一个名为Apps的新文件夹，其中包含你的{New App}文件夹。其中应该包含文件和数据库的备份文件夹。因此，对于名为erpnext的应用，以下是文件夹位置：

Database files: /Apps/erpnext/database
Public files: /Apps/erpnext/files
Private files: /Apps/erpnext/private/files

注意：如果压缩备份大小超过 1GB，系统将上传最新的可用备份到 Dropbox，而不是生成新的备份文件。
