# 物料需求计划（MRP）

ERPNext 目前将 MRP 集成在生产计划中，这对新用户来说难以理解，而且它仅使用销售订单或物料请求数据来生成 MRP。

目的

MRP 的主要目的是确保正确的材料在正确的时间可用，帮助公司避免生产过剩和交付延迟。要了解 MRP 的工作原理，首先需要理解需求规划是如何运作的。

需求规划

我们已经拥有销售订单和物料请求来生成需求规划。然而，这仅适用于短期 MRP 需求，无法覆盖较长时间范围。为了解决这一问题，我们添加了销售预测和交付计划来生成需求。

销售预测

我们使用 Holt-Winters 方法基于过去的销售订单数据进行销售预测。Holt-Winters 预测方法同时考虑趋势和季节性因素。但使用 Holt-Winters 进行预测并非强制要求。用户也可以通过在系统外部计算后手动输入预测数据。

!图片

交付计划

一些客户不喜欢针对销售订单进行一次性交付，而是需要多次交付。交付计划对于 MRP 非常有帮助，因为它们提供了材料实际需求时间的可见性，而不是假设所有货物必须一次性交付。 要创建交付计划，用户需要在销售订单明细表中点击"添加计划"按钮 !图片

主生产计划

为了捕获成品的需求，引入了主生产计划（MPS）文档。系统从各种来源收集数据，如销售订单、物料请求、销售预测和交付计划，以在 MPS 中生成需求。

!图片

物料需求计划报告

物料需求计划（MRP）报告根据成品的需求计算原材料和零部件的所需数量，以及它们的需求日期。

项目提前期

为了计算发布原材料采购订单和零部件工作订单的日期，系统需要了解制造提前期和材料采购提前期。为此，添加了一个新文档：项目提前期。
