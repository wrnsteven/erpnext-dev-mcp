# 销售设置

在这里可以定义将应用于销售周期中涉及的主数据和交易的属性和验证规则。

要访问销售设置，请前往：

> 首页 > 销售 > 设置 > 销售设置

客户默认值

!图片

1. 客户命名方式

保存客户时，系统会为该客户生成一个唯一ID。

默认情况下，客户ID根据客户名称生成。

如果您希望使用命名序列来保存客户，请在"客户命名序列"字段中将值设置为"命名序列"。使用命名序列保存的客户ID示例 - "CUST00001, CUST00002, CUST00003..." 以此类推。

2. 默认客户组

选择一个默认客户组，创建新客户时将自动更新该字段。

报价单可以针对客户创建，也可以针对线索创建。当将针对线索创建的报价单转换为销售订单时，系统会尝试将该线索转换为客户。在后端创建客户时，"客户组"的值从销售设置中获取。如果未找到客户组的默认值，则会收到一条验证消息，要求提供客户组。您也可以手动将线索转换为客户。

3. 默认区域

选择一个默认区域，创建新客户时将自动更新该字段。

报价单可以针对客户创建，也可以针对线索创建。当将针对线索创建的报价单转换为销售订单时，系统会尝试将该线索转换为客户。在后端创建客户时，"区域"的值从销售设置中获取。如果未找到区域的默认值，则会收到一条验证消息，要求提供区域。您也可以手动将线索转换为客户。

产品价格设置

!图片

1. 默认价格列表

此字段中设置的价格列表将自动更新到报价单、销售订单、交货单和销售发票等销售交易的价格列表字段中。

2. 全销售周期保持相同单价

确保销售流程中保持相同的单价。启用后，ERPNext 将验证从销售订单创建交货单或销售发票时，产品的价格是否发生变化。

3. 使用默认价格列表的价格作为备选

如果在交易中设置的价格列表中找不到产品的产品报价，则将从默认价格列表获取价格。请勿与库存设置中的"自动插入缺失的产品报价"一起使用，因为默认价格列表的价格可能会出现在其他价格列表中。
