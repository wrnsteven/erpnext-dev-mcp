# 页面和报告的角色权限

可以通过"页面和报告的角色权限"控制对不同页面和报告的访问权限。

文档类型包括销售订单、客户、供应商等。它们是一种文档类型，意味着可以包含多个该类型的文档。页面就像销售设置一样，是一个单独的页面。你无法创建多个销售设置，但可以创建多个销售订单。

在 ERPNext 中，用户可以使用页面创建自定义用户界面，或使用报表构建器或查询报表创建自定义报表。ERPNext 有一个基于角色的权限系统，你可以为用户分配角色。同一角色可以分配给页面和报告以便访问它们。

如果用户已启用开发者模式，则可以直接在页面和报告记录中添加角色。在这种情况下，权限也会反映在页面/报告的 JSON 文件中。假设你想限制可以访问 ERPNext 中某些页面和报告的角色，可以通过页面和报告的角色权限来实现。

要访问页面和报告的角色权限，请转到：

> 首页 > 用户和权限 > 页面和报告的角色权限

1. 如何使用页面和报告的角色权限工具

如果开发者模式被禁用，用户可以使用"页面和报告的角色权限"页面为页面和报告分配角色。

1.1 重置为默认值

使用"重置为默认值"按钮，用户可以移除在页面或报告上应用的自定义权限。然后该页面或报告将适用默认权限。
