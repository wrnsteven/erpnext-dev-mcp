添加用户

ERPNext 中的用户管理包括创建和管理用户账户、定义角色以及根据用户的职责分配角色。管理员可以配置每个角色的详细权限，指定他们对系统内模块的访问级别。

用户可以由 System Manager 添加。添加用户请前往: > Home > Users and Permissions > User

用户主要有两种类型:

网站用户: 客户、供应商、学生等，只能访问门户，不能访问任何模块。系统用户: 在公司使用 ERPNext 的人员，可以访问模块、公司数据等。

了解更多关于 系统用户和网站用户的区别。

在用户文档中，可以输入很多信息。为了便于使用，为网络用户输入的信息很少: 姓名和电子邮件。

电子邮件地址是识别用户的唯一键( ID )。

1. 创建新用户

  1. 转到用户列表，点击"新建"。
  2. 添加用户的电子邮件地址和姓名。
  3. 保存。

也可以更改用户名和语言等详细信息。

2. 功能

2.1 设置角色

保存后，您将看到角色列表及其旁边的复选框。只需勾选您希望用户拥有的角色，然后保存文档。角色具有预定义的权限，了解更多关于角色的信息，请点击此处。您可以设置 角色配置文件 作为模板，一次选择多个角色。

2.2 更多信息

可以在此部分设置有关员工的更多信息:

  • Gender
  • Phone
  • Mobile No
  • Birth Date
  • Location
  • Interests
  • Bio

勾选"静音"将静音与文档交互时播放的声音。用户可能需要执行设置 > 重新加载才能使更改生效。

2.3 修改密码

  • 设置新密码: 作为 System Manager，您可以根据需要为用户设置新密码。
  • 发送密码更新通知: 向用户发送密码已更改的电子邮件通知。
  • 修改密码时从所有设备登出: 修改用户密码时，这会使该用户从 PC 和可能登录的任何移动设备登出。

2.4 文档关注

通过此选项，您可以关注系统中的各种文档，并在更新时收到电子邮件通知。在此处了解更多。

2.5 电子邮件设置

  • 发送电子邮件线程通知: 用户将收到在 Opportunities 等文档类型中进行的电子邮件对话通知。
  • 发送外发电子邮件副本: 向用户发送他们发送的电子邮件副本。这对于跟踪电子邮件是否已发送很有用。
  • 允许提及: 允许此用户的姓名出现在线程对话中，以便可以使用 '@' 来提及他们。
  • 电子邮件签名: 在此处添加电子邮件签名将作为该用户所有外发电子邮件的默认签名。这与从 公司主数据 设置的页脚不同。

2.6 电子邮件收件箱

从此部分订阅公司不同邮件列表的用户。添加新行并选择邮件列表以分配给此用户。例如，邮件列表可以是 jobs、support、sales 等。了解更多关于 Email Inbox 的信息，请点击此处。

2.7 允许模块访问

用户将可以访问他们拥有基于角色访问权限的所有模块。如果你想限制此用户对某些模块的访问，请从此列表中取消勾选模块。

2.7.1 模块配置文件

角色配置文件作为模板，用于存储和选择对多个模块的访问权限。然后可以将此角色配置文件分配给用户。例如，HR 用户将可以访问 HR、Payroll 等多个模块。角色配置文件有助于在添加多个用户时一次提供对多个模块的访问权限。

2.8 安全设置

  • 同时会话: 允许用户的同时登录会话数。您可以通过允许更多会话为多个用户使用同一组凭据。这可以在 系统设置 中全局限制。对于云账户，同时会话总数不能超过订阅用户总数。
  • 用户类型: 如果用户勾选了 Customer、Supplier、Patient 或 Student 以外的角色，他们将自动成为系统用户。此字段为只读。
  • 登录时间、登录前: 如果您希望仅在办公时间内或在周末授予用户系统访问权限，请在此处指定。例如，如果办公时间是从上午 10 点到下午 6 点，则将登录时间、登录前小时设置为 10:00 和 18:00。
  • 限制 IP: 将用户登录限制在此处指定的 IP。这可用于使用户只能从办公计算机登录。多个 IP 可以用逗号分隔添加。

此部分还显示用户的其他详细信息，如上次登录、上次 IP 和上次活跃时间。

2.9 第三方认证

这将允许用户使用 Facebook、Google 或 GitHub 登录。要使用此功能，请在 Facebook、Google、GitHub 等处注册开发者账户。在他们的控制台创建一个应用，指定应用名称、来源 URL 和回调 URL，复制客户端 ID 和客户端密钥信息到此处以开始使用。

更多详情，请转到 此页面。

2.10 API 访问

您可以使用"生成密钥"按钮从此部分生成 API 密钥。这可用于从另一个应用程序访问您账户的数据，例如离线 POS 系统。

2.11 保存后

保存用户后，您将在用户主数据的仪表板区域看到这些按钮。

权限

  • 设置用户权限: 将带您进入 Bruce 的 用户权限 页面，您可以在其中限制 Bruce 对文档的访问权限。
  • 查看允许的文档: 将带您进入该用户的"用户允许文档"报告。在这里您可以看到 Bruce 可以访问哪些文档。例如，在选定的销售订单上，将显示 Bruce 可以访问的销售订单列表。

密码

  • 重置密码: 包含重置用户密码说明的电子邮件将发送到用户的 电子邮件账户。
  • 重置 OTP 密钥: 重置通过双因素身份验证登录的 OTP 密钥。

创建用户电子邮件将使您能够基于在用户主数据中输入的电子邮件为用户创建 电子邮件账户。

模拟

版本 15 中的新功能。

有时，为了解用户面临的特定问题，您可能需要以该用户身份登录。管理员用户可以使用用户文档上的"模拟"按钮模拟任何其他用户。

注意:

  • 每当管理员使用"模拟"功能时，都会创建活动日志。
  • 任何启用了跟踪更改的文档更改也会指出在修改文档时模拟他人的原始用户。
  • 您模拟的用户将收到通知。
