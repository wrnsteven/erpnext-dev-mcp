# 销售退货

**已售商品被退回称为销售退货。**

企业经常会有已售商品的退货。客户可能因为质量问题、未按约定日期交付或其他原因退货。

## 前置条件

在创建和使用销售退货之前，建议先创建以下内容：

- Item（商品）
- Sales Invoice 或 Delivery Note（销售发票或交货单）

## 如何创建销售退货

首先打开原始的 Delivery Note / Sales Invoice（交货单/销售发票），即客户退货所对应的原始单据。

Original Delivery Note（原始交货单）

然后点击 **'Create > Sales Return'**（创建 > 销售退货），系统将打开一个新的 Delivery Note（交货单），其中 **'Is Return'**（是否退货）已被勾选，商品、单价和税费的数值将显示为负数。

Return Against Delivery Note（针对交货单的退货）

您也可以针对原始 Sales Invoice（销售发票）创建退货单据，若要在退货时同时退回库存，请在 Return Sales Invoice（退货销售发票）中勾选 **"Update Stock"**（更新库存）选项。

Return Against Sales Invoice（针对销售发票的退货）

提交 Return Delivery Note / Sales Invoice（退货交货单/销售退货发票）后，系统将增加指定 Warehouse（仓库）的库存余额。为保持正确的库存估值，库存余额将按照退货商品的原始采购单价增加。

Return Stock Ledger（退货库存账本）

在 Return Sales Invoice（退货销售发票）的情况下，客户账户将被贷记，相关的收入账户和税费账户将被借记，如会计账本所示。

Return Stock Ledger（退货库存账本）

如果启用了 Perpetual Inventory（永续库存），系统还将过账针对仓库账户的会计分录，以使仓库账户余额与库存账本中的库存余额保持一致。

## 通过交货单退货对库存的影响

针对 Delivery Note（交货单）创建销售退货时：

原始 Delivery Note（交货单）及其关联的任何 Sales Order（销售订单）中的 **Returned Quantity**（已退货数量）将被更新。

如果 100% 已退货，原始 Delivery Note（交货单）的状态将变更为 **Return Issued**（已发出退货）：
