可以通过基于角色的权限来控制对不同文档的访问权限。

ERPNext 采用基于角色的权限系统。这意味着你可以为用户分配角色，并可以在角色上设置权限。角色权限管理器允许你设置哪些角色可以访问哪些文档，以及具有哪些权限（读取、写入、提交等）。

一旦为用户分配了角色，他们的访问权限可以限制为特定文档。权限结构允许你使用称为字段**权限级别**的概念为不同字段定义不同的权限规则。

- 如何使用角色权限管理器

要开始使用角色权限管理器，请访问：> 首页 > 用户和权限 > 角色权限管理器

![图片](../../images/382_setting-up-permissions-leave-application.png)

权限应用于以下组合：

- **角色：** 正如我们之前看到的，用户被分配角色，权限规则是在这些角色上应用的。例如，销售用户可能被分配员工和销售用户角色。

角色示例包括账务经理、员工、人力资源用户等。

- **文档类型：** 每种文档类型，无论是主数据还是交易，都有单独的角色权限列表，如前面的截图所示。

文档类型示例包括销售发票、请假申请、库存录入等。

- **权限级别：** 在每个文档中，你可以将字段按"级别"分组。每个字段组用一个唯一数字（0 到 9）表示。可以为每个字段组应用单独的权限规则集。默认情况下，所有字段都属于级别 0。

权限"级别"将具有级别 X 的字段与级别 X 的权限规则相关联。了解更多请点击[此处](/erpnext/changing-the-properties-of-a-field-based-on-role)。

- **文档阶段：** 权限应用于文档的每个阶段，如创建、保存、提交、取消和修订。角色可以被授权打印、发送电子邮件、导入或导出数据、访问报表或定义用户权限。

- **用户权限：** 在 ERPNext 中使用用户权限，可以限制用户只能访问该文档类型的特定文档。例如：只能访问所有地区中的一个地区。如果用户权限与其他文档类型相关联（通过链接字段），它们也会应用于当前文档类型。

例如，客户是销售订单或报价单中的链接字段。在角色权限管理器中，可以使用"设置用户权限"按钮设置用户权限。

要基于文档/字段设置用户权限，请访问：> 首页 > 用户和权限 > 权限 > [用户权限](/erpnext/user-permissions)

- **添加新规则**：在角色权限管理器中，要添加新规则，请点击**添加新规则**按钮，弹出框会提示你选择角色和权限级别。一旦选择并点击"添加"，这将在规则表中新增一行。

- 基于角色的权限如何工作

请假申请是一个涵盖权限系统所有方面的良好示例。

- 它应该由员工创建。为此，员工角色应被授予读取、写入、创建权限。

![图片](../../images/382_setting-up-permissions-employee-role.png)

- 员工应该只能访问他/她的请假申请。因此，应为每个用户-员工组合创建用户权限记录。

![图片](../../images/382_setting-up-permissions-employee-user-permissions.png)

- 如果你希望员工仅能在另一个文档中选择某个文档，而不完全读取该文档，则仅授予员工角色选择权限。

![图片](../../images/382_setting-up-select-permissions-employee.png)

- **人力资源经理**应该能够查看所有请假申请。在级别 0 为人力资源经理创建权限规则，授予读取权限。禁用"应用用户权限"选项。

![图片](../../images/382_setting-up-permissions-hr-manager-role.png)

- **请假审批人**应该能够查看和更新其下属员工的请假申请。在级别 0 授予请假审批人读取和写入权限。相关的员工文档应列入请假审批人的用户权限中。（对于员工文档中提到的请假审批人，系统会通过编程方式自动创建用户权限记录，从而减少此工作量）。

![图片](../../images/382_setting-up-permissions-leave-approver-role.png)

- 请假申请只能由人力资源用户或请假审批人批准/拒绝。请假申请的状态字段位于级别 1。人力资源用户和请假审批人在级别 0 被授予读取和写入权限，而其他所有人（所有人）在级别 1 只有读取权限。

![图片](../../images/382_setting-up-permissions-level-1.png)

- **人力资源用户**应该能够将请假申请委托给他/她的下属。人力资源用户被授予设置用户权限的权利。具有人力资源用户角色的用户能够为其他用户定义请假申请上的用户权限。

![图片](../../images/382_setting-up-permissions-hr-user-role.png)

如果你已正确分配了角色，但在访问文档时仍然遇到错误，请参阅[此页面](/erpnext/report-permission-error)。

### 3. 权限类型

在角色权限管理器中，可以为每个角色启用多个权限。以下是可用权限选项及其用法的详细信息。

![图片](../../images/382_Screenshot%202025-12-19%20at%205.15.56%E2%80%AFPM.png)

| 权限 | 描述 | 示例 |
|------|------|------|
| Select | 允许用户搜索和查看记录。 | 用户可以在销售订单中选择客户，但无法打开客户主数据。 |
| Read | 允许用户查看文档。 | 用户可以查看销售发票，但无法更改它们。 |
| Write | 允许用户编辑他们有权访问的现有记录。 | 用户可以更新客户地址，但无法创建新客户。 |
| Create | 允许用户创建新文档。 | 用户可以创建新物料，但无法编辑现有物料。 |
| Delete | 允许用户删除文档。 | 用户可以删除已提交文档。 |
| Print | 允许打印或下载文档的 PDF。 | 文档中的打印按钮对该用户可用。 |
| Email | 允许用户从文档发送电子邮件。 | 文档中的电子邮件按钮对该用户可用。 |
| Report | 允许用户访问与文档相关的报表。 | 如果用户有权访问员工且启用了报表，他们可以查看基于员工的报表。 |
| Export | 允许用户从报表视图导出数据。 | 用户可以导出报表数据。 |
| Import | 允许用户向文档导入数据。 | 用户可以为文档导入数据。 |
| Share | 允许与其他用户共享文档访问权限。 | 用户可以与其他用户共享文档访问权限。 |
| Mask（数据掩码） | 在 ERPNext Version 16 中，数据掩码允许根据用户的角色和权限配置特定字段显示掩码或隐藏值，而不影响字段可见性。确保敏感信息得到保护，同时保持用户界面的一致性和信息完整性。 |
