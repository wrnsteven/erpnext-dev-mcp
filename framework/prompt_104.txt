            A DocField defines a property (or a field) of a DocType. You can define the column name, label, datatype and more for DocFields. For instance, a ToDo doctype has fields `description`, `status` and `priority`. These ultimately become columns in the database table `tabToDo`.

Example

The DocField stores meta-data about the field. Some of them are described below.

`[
 {
 &quot;label&quot;: &quot;Description&quot;, // the value shown to the user (Form, Print, etc)
 &quot;fieldname&quot;: &quot;description&quot;, // the property name we refer in code, also the column name
 &quot;fieldtype&quot;: &quot;Text Editor&quot;, // the fieldtype which also decides how to store this value
 &quot;reqd&quot;: 1 // whether this field is mandatory
 },
 {
 &quot;label&quot;: &quot;Status&quot;,
 &quot;fieldname&quot;: &quot;status&quot;,
 &quot;fieldtype&quot;: &quot;Select&quot;,
 &quot;options&quot;: [
 &quot;Open&quot;,
 &quot;Pending&quot;,
 &quot;Closed&quot;
 ]
 },
 {
 &quot;label&quot;: &quot;Priority&quot;,
 &quot;fieldname&quot;: &quot;priority&quot;,
 &quot;fieldtype&quot;: &quot;Select&quot;,
 &quot;options&quot;: [ // list of options for select
 &quot;Low&quot;,
 &quot;Medium&quot;,
 &quot;High&quot;
 ],
 &quot;default&quot;: &quot;Low&quot; // the default value to be set
 },
 {
 &quot;label&quot;: &quot;Completed By&quot;,
 &quot;fieldname&quot;: &quot;completed_by&quot;,
 &quot;fieldtype&quot;: &quot;Link&quot;,
 &quot;options&quot;: &quot;User&quot;,
 &quot;depends_on&quot;: &quot;eval: doc.status == 'Closed'&quot;, // the condition on which this field's display depends
 },
 {
 &quot;collapsible&quot;: 1,
 &quot;collapsible_depends_on&quot;: &quot;eval:doc.status!='Closed'&quot;, // determines if a Section Break field is collapsible
 &quot;fieldname&quot;: &quot;sb_details&quot;,
 &quot;fieldtype&quot;: &quot;Section Break&quot;,
 &quot;label&quot;: &quot;Details&quot;
 },
 {
 &quot;fieldname&quot;: &quot;amount&quot;,
 &quot;fieldtype&quot;: &quot;Currency&quot;, // Currency field
 &quot;label&quot;: &quot;Amount&quot;,
 &quot;non_negative&quot;: 1, // determines whether this field value can be negative
 &quot;options&quot;: &quot;INR&quot;,
 }
]
`
Similar to the `depends_on` property which determines whether a field will be displayed or not,
in Version 12 we have introduced two new properties:

`mandatory_depends_on`: If this condition is satisfied, the field will be mandatory.

- `read_only_depends_on`: If this condition is satisfied, the field will be read only.

Frappe comes with more than 30 different fieldtypes out-of-the-box.
These fieldtypes serve a variety of use-cases. You can learn more about fieldtypes in the next page.
