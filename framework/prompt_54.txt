            This guide discusses how to add a social login provider to frappe via pull request.

### Add your provider in `SocialLoginKey.get_social_login_provider`

`providers[&quot;Frappe&quot;] = {
 &quot;provider_name&quot;: &quot;Frappe&quot;,
 &quot;enable_social_login&quot;: 1,
 &quot;custom_base_url&quot;: 1,
 &quot;icon&quot;:&quot;/assets/frappe/images/favicon.png&quot;,
 &quot;redirect_url&quot;: &quot;/api/method/frappe.www.login.login_via_frappe&quot;,
 &quot;api_endpoint&quot;: &quot;/api/method/frappe.integrations.oauth2.openid_profile&quot;,
 &quot;api_endpoint_args&quot;:None,
 &quot;authorize_url&quot;: &quot;/api/method/frappe.integrations.oauth2.authorize&quot;,
 &quot;access_token_url&quot;: &quot;/api/method/frappe.integrations.oauth2.get_token&quot;,
 &quot;auth_url_data&quot;: json.dumps({
 &quot;response_type&quot;: &quot;code&quot;,
 &quot;scope&quot;: &quot;openid&quot;
 })
}
`
### Add provider key in exact same type case in options of `social_login_provider` select field on `Social Login Key` DocType. e.g. `Frappe`

Once the user adds a social login provider and enables it the `Authorization Code` is sent back by the provider api server on to the redirect_url mentioned on the same server. You will have to add a whitelisted method allowing guest access in `frappe.integrations.oauth2_logins`. e.g. `login_via_office365`

There many implementations of OAuth 2.0 + OpenID Connect. Here we'll discuss two ways of accessing openid information.

#### User Creation via OpenID Profile Endpoint

example:

`@frappe.whitelist(allow_guest=True)
def login_via_frappe(code, state):
 login_via_oauth2(&quot;frappe&quot;, code, state, decoder=json.loads)
`
#### User Creation via id_token

example:

`@frappe.whitelist(allow_guest=True)
def login_via_office365(code, state):
 login_via_oauth2_id_token(&quot;office_365&quot;, code, state, decoder=json.loads)
`
