            Here, let's take a look into how responses are built in Frappe, and how you may
be able to use them in your Frappe apps or scripts.

If you have already gone through the [Router
Documentation](/framework/v14/user/en/python-api/routing-and-rendering), you might've noticed the
`build_response` function that Frappe internally utilizes to build responses
depending on the type of the content. The logic that defines this behaviour is a
part of the module
[frappe.utils.response](https://github.com/frappe/frappe/blob/develop/frappe/utils/response.py),
of which `build_response` is the meat and potatoes.

`def build_response(response_type=None):
 if &quot;docs&quot; in frappe.local.response and not frappe.local.response.docs:
 del frappe.local.response[&quot;docs&quot;]

 response_type_map = {
 &quot;csv&quot;: as_csv,
 &quot;txt&quot;: as_txt,
 &quot;download&quot;: as_raw,
 &quot;json&quot;: as_json,
 &quot;pdf&quot;: as_pdf,
 &quot;page&quot;: as_page,
 &quot;redirect&quot;: redirect,
 &quot;binary&quot;: as_binary,
 }

 return response_type_map[frappe.response.get(&quot;type&quot;) or response_type]()
`
The above snippet represents the current implementation of `build_response`
which maps different functions that act as handlers for different content types.
Let's take a deeper look into the response handler for the &quot;download&quot;
`response_type` in Frappe v13.

`def as_raw():
 response = Response()
 response.mimetype = (
 frappe.response.get(&quot;content_type&quot;)
 or mimetypes.guess_type(frappe.response[&quot;filename&quot;])[0]
 or &quot;application/unknown&quot;
 )
 response.headers[&quot;Content-Disposition&quot;] = (
 f'{frappe.response.get(&quot;display_content_as&quot;, &quot;attachment&quot;)};'
 f' filename=&quot;{frappe.response[&quot;filename&quot;].replace(&quot; &quot;, &quot;_&quot;)}&quot;'
 ).encode(&quot;utf-8&quot;)
 response.data = frappe.response[&quot;filecontent&quot;]
 return response
`
Depending on the value of the `Content-Disposition` header, the browser
receiving the response may behave differently. If unset, the value defaults to
&quot;attachment&quot;.

If `frappe.response.display_content_as` is set to &quot;inline&quot;, it indicates
that the content is expected to be displayed inline in the browser, that is, as
a Web page or as part of a Web page, while &quot;attachment&quot; means the contents
are to be downloaded and saved locally.

To create an API endpoint that would directly download the file you require, you
could craft something like the following to download the file directly.

`@frappe.whitelist()
def download(name):
 file = frappe.get_doc(&quot;File&quot;, name)
 frappe.response.filename = file.file_name
 frappe.response.filecontent = file.get_content()
 frappe.response.type = &quot;download&quot;
 frappe.response.display_content_as = &quot;attachment&quot;
`
